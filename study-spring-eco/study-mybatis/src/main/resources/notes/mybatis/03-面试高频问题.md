# MyBatis 面试高频问题

## 问题 1：MyBatis 执行流程（从 Mapper 接口调用到结果返回）

**完整 13 步调用链：**

```
MapperProxy.invoke()                    -- JDK 动态代理拦截接口方法
    ↓
MapperMethod.execute()                  -- 根据 SqlCommandType 路由
    ↓
SqlSession.selectOne() / selectList()   -- 门面模式，委托给 Executor
    ↓
CachingExecutor.query()                 -- 检查二级缓存
    ↓
BaseExecutor.query()                    -- 检查一级缓存
    ↓
BaseExecutor.queryFromDatabase()        -- 缓存未命中，查库
    ↓
SimpleExecutor.doQuery()                -- 创建 StatementHandler
    ↓
Configuration.newStatementHandler()     -- RoutingStatementHandler → PreparedStatementHandler
    ↓
StatementHandler.parameterize()         -- 委托 ParameterHandler 设置参数
    ↓
ParameterHandler.setParameters()        -- TypeHandler Java→JDBC 类型转换
    ↓
StatementHandler.query()                -- 执行 PreparedStatement.executeQuery()
    ↓
ResultSetHandler.handleResultSets()     -- ResultSet → Java 对象映射（TypeHandler）
    ↓
返回结果给调用方                        -- 结果返回 MapperProxy → 业务代码
```

**关键设计模式：**
- 构建者模式：SqlSessionFactoryBuilder
- 工厂模式：SqlSessionFactory
- 门面模式：SqlSession
- 代理模式：MapperProxy
- 模板方法模式：BaseExecutor
- 装饰器模式：CachingExecutor
- 策略模式：StatementHandler
- 责任链模式：InterceptorChain

**相关问题：**
- MyBatis 支持哪些 Executor？（SimpleExecutor、ReuseExecutor、BatchExecutor）
- 如何切换 Executor 类型？（settings 中配置 defaultExecutorType）

## 问题 2：一级缓存 vs 二级缓存

| 维度 | 一级缓存 | 二级缓存 |
|------|----------|----------|
| 级别 | SqlSession 级别 | Mapper (namespace) 级别 |
| 默认状态 | 开启，不可关闭 | 关闭，需手动配置 `<cache/>` |
| 作用域 | 同一个 SqlSession 内 | 跨 SqlSession（同一 namespace） |
| 存储 | HashMap（内存） | 可插拔（默认 PerpetualCache，支持 Ehcache/Redis） |
| 命中条件 | 同 Session + 同 SQL + 同参数 | 跨 Session + 同 namespace + 同 SQL + 同参数 |
| 清空时机 | commit / close / insert/update/delete | 同一 namespace 内 insert/update/delete |
| 序列化 | 不需要 | 实体必须实现 Serializable |
| 数据一致性 | 强（Session 内） | 弱（分布式环境可能不一致） |

**一级缓存命中示例：**
```java
SqlSession session = factory.openSession();
UserMapper mapper = session.getMapper(UserMapper.class);

User user1 = mapper.selectById(1L); // 查数据库，放入一级缓存
User user2 = mapper.selectById(1L); // 一级缓存命中，不再查库
// user1 == user2 为 true（同一对象引用）

session.close(); // 一级缓存销毁
```

**一级缓存失效（脏读问题）：**
```java
SqlSession session1 = factory.openSession();
SqlSession session2 = factory.openSession();

UserMapper mapper1 = session1.getMapper(UserMapper.class);
UserMapper mapper2 = session2.getMapper(UserMapper.class);

User user1 = mapper1.selectById(1L); // session1 查库，缓存
mapper2.updateById(new User(1L, "新名字", 30)); // session2 更新，提交
User user2 = mapper1.selectById(1L); // session1 一级缓存命中 → 读到旧数据（脏读！）
```

**二级缓存跨 SqlSession：**
```java
SqlSession session1 = factory.openSession();
UserMapper mapper1 = session1.getMapper(UserMapper.class);
User user1 = mapper1.selectById(1L); // 查库 → 一级缓存 + 二级缓存
session1.close(); // 一级缓存写入二级缓存

SqlSession session2 = factory.openSession();
UserMapper mapper2 = session2.getMapper(UserMapper.class);
User user2 = mapper2.selectById(1L); // 二级缓存命中！不查库
session2.close();
```

## 问题 3：#{} vs ${} + SQL 注入防御

### 核心区别

| 维度 | #{ } | ${ } |
|------|-------|------|
| 底层实现 | PreparedStatement `?` 占位符 | 字符串直接拼接 |
| 编译时机 | 预编译（SQL 缓存） | 运行时拼接（无法缓存） |
| SQL 注入 | **安全**，参数值不会被当作 SQL | **危险**，参数值被当作 SQL 一部分 |
| 类型处理 | TypeHandler 自动类型转换 | String 拼接 |
| 单引号 | 自动添加（字符串类型） | 需手动添加 |
| 适用场景 | WHERE 条件值、INSERT/UPDATE 值 | 表名、列名、ORDER BY、动态 SQL 片段 |

### #{ } 工作原理

```sql
-- XML 配置
SELECT * FROM user WHERE id = #{id} AND name = #{name}

-- 解析后
SELECT * FROM user WHERE id = ? AND name = ?

-- JDBC 执行
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setLong(1, 1L);
pstmt.setString(2, "张三");
ResultSet rs = pstmt.executeQuery();
-- 数据库接收到的：SELECT * FROM user WHERE id = 1 AND name = '张三'
```

### ${ } 工作原理

```sql
-- XML 配置
SELECT * FROM user WHERE name = '${name}'

-- 解析后（name="张三"）
SELECT * FROM user WHERE name = '张三'

-- JDBC 执行（Statement，非 PreparedStatement）
Statement stmt = conn.createStatement();
ResultSet rs = stmt.executeQuery(sql);
```

### SQL 注入攻击演示（${ } 的风险）

```sql
-- 恶意输入：' OR '1'='1
-- 拼接后 SQL：
SELECT * FROM user WHERE name = '' OR '1'='1'
-- 结果：返回全部用户数据！

-- 恶意输入：'; DROP TABLE user; --
-- 拼接后 SQL：
SELECT * FROM user WHERE name = ''; DROP TABLE user; --'
-- 结果：user 表被删除！

-- 恶意输入：' UNION SELECT id, password FROM admin --
-- 拼接后 SQL：
SELECT * FROM user WHERE name = '' UNION SELECT id, password FROM admin --'
-- 结果：admin 表密码泄露！
```

### ${ } 的安全使用场景

```xml
<!-- 安全的 ${ } 使用：表名/列名 动态切换（需白名单校验） -->
<select id="selectByOrder">
    SELECT * FROM user ORDER BY ${orderColumn} ${orderDirection}
</select>

<!-- 前端传入 orderColumn=id, orderDirection=DESC → ORDER BY id DESC -->
<!-- 需要后端白名单校验，防止 orderColumn=evil_column  -->
```

## 问题 4：MyBatis 插件原理

### 插件接口

```java
public interface Interceptor {
    Object intercept(Invocation invocation) throws Throwable;
    default Object plugin(Object target) { return Plugin.wrap(target, this); }
    default void setProperties(Properties properties) { }
}
```

### 可拦截的四大对象

| 拦截对象 | 可拦截方法 | 典型插件 |
|----------|------------|----------|
| Executor | update, query, flushStatements, commit, rollback, getTransaction, close, isClosed | PageHelper（分页）、性能监控 |
| StatementHandler | prepare, parameterize, batch, update, query | 多数据源路由、SQL 改写 |
| ParameterHandler | getParameterObject, setParameters | 参数加密/脱敏 |
| ResultSetHandler | handleResultSets, handleOutputParameters | 结果集加密/脱敏 |

### Plugin.wrap() 源码解析

```java
// Plugin 实现 InvocationHandler
public class Plugin implements InvocationHandler {
    private final Object target;
    private final Interceptor interceptor;
    private final Map<Class<?>, Set<Method>> signatureMap;

    // 生成代理对象
    public static Object wrap(Object target, Interceptor interceptor) {
        Map<Class<?>, Set<Method>> signatureMap = getSignatureMap(interceptor);
        Class<?> type = target.getClass();
        Class<?>[] interfaces = getAllInterfaces(type, signatureMap);
        if (interfaces.length > 0) {
            return Proxy.newProxyInstance(
                type.getClassLoader(), interfaces,
                new Plugin(target, interceptor, signatureMap));
        }
        return target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        Set<Method> methods = signatureMap.get(method.getDeclaringClass());
        if (methods != null && methods.contains(method)) {
            // 匹配到 @Signature 声明的方法 → 走拦截器
            return interceptor.intercept(new Invocation(target, method, args));
        }
        // 不匹配 → 直接调用原方法
        return method.invoke(target, args);
    }
}
```

### 分页插件（PageHelper）原理

```
1. 拦截 Executor.query(MappedStatement, parameter, RowBounds, ResultHandler)
2. 检查 PageHelper.startPage() 设置的 ThreadLocal 是否有分页参数
3. 有分页参数：
   a. 先执行 COUNT SQL：SELECT COUNT(*) FROM (原始SQL) tmp_count
   b. 设置 total 到 Page 对象
   c. 改写原始 SQL 追加 LIMIT（根据数据库方言）
   d. 执行改写后的 SQL 获取分页数据
   e. 将结果设置到 Page.records
4. 返回 Page 对象（继承自 ArrayList）
```

## 问题 5：MyBatis vs MyBatis Plus vs JPA/Hibernate

| 维度 | MyBatis | MyBatis Plus | JPA/Hibernate |
|------|---------|--------------|---------------|
| ORM 类型 | 半自动 SQL Mapping | 半自动（增强 MyBatis） | 全自动 ORM |
| SQL 控制 | 完全手写 SQL | 手写 + 自动生成 | 自动生成（JPQL/HQL） |
| CRUD 效率 | 需手写 SQL/XML | BaseMapper 开箱即用 | JpaRepository 开箱即用 |
| 复杂查询 | 灵活强大 | Wrapper 条件构造器 | Criteria API / @Query |
| 动态 SQL | XML `<if>/<foreach>` 等 | Wrapper 链式调用 | Specification / CriteriaBuilder |
| 分页 | 手动 LIMIT | Page + 插件自动 | Pageable 自动分页 |
| 乐观锁 | 手动 version 字段 | @Version 注解自动 | @Version 注解自动 |
| 逻辑删除 | 手动标记字段 | @TableLogic 注解 | @SQLDelete + @Where |
| 缓存 | 一级 + 二级缓存 | 一级 + 二级缓存 | 一级 + 二级 + 查询缓存 |
| 学习曲线 | 中等 | 低 | 高（JPA 规范 + Hibernate） |
| 性能调优 | SQL 直接可调优 | SQL 直接可调优 | 需分析自动生成 SQL |
| 数据库移植 | 手动适配 SQL 方言 | 手动适配 SQL 方言 | Dialect 自动适配 |
| N+1 问题 | 手写 SQL 可控 | 手写 SQL 可控 | 需配置 @Fetch 或 JOIN FETCH |
| 延迟加载 | 通过 association/collection | 通过 association/collection | 默认懒加载（需事务内） |

### 选型建议

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 复杂 SQL 多、高性能要求 | MyBatis / MyBatis Plus | 直接控制 SQL，调优空间大 |
| 快速开发、标准化 CRUD 多 | MyBatis Plus / Spring Data JPA | 自动 CRUD，减少样板代码 |
| 微服务架构、快速原型 | Spring Data JPA | 约定优于配置，开发效率高 |
| 大型互联网项目（高并发 + 复杂查询） | MyBatis Plus | 兼顾开发效率和 SQL 控制力 |
| 需要完全控制 SQL 的业务 | MyBatis（原生） | 最细粒度的 SQL 控制 |
| 数据库迁移频繁 | JPA/Hibernate | Dialect 自动适配，代码无需改动 |
