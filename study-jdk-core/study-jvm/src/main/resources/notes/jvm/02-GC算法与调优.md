## 3. 四种引用

```mermaid
graph LR
    Strong["强引用<br/>Strong Reference<br/>永不回收<br/><code>Object obj = new Object()</code>"]
    Soft["软引用<br/>Soft Reference<br/>内存不足时回收<br/><code>new SoftReference(obj)</code>"]
    Weak["弱引用<br/>Weak Reference<br/>下次 GC 回收<br/><code>new WeakReference(obj)</code>"]
    Phantom["虚引用<br/>Phantom Reference<br/>回收时通知<br/><code>new PhantomReference(obj, q)</code>"]

    Strong -->|"引用强度递减"| Soft --> Weak --> Phantom
```

| 引用类型 | 回收时机 | get() 行为 | 典型场景 |
|----------|----------|-----------|----------|
| 强引用 | 永不回收 | 正常返回 | 99% 的对象 |
| 软引用 | 内存不足（即将 OOM） | GC 前返回对象 | 缓存（图片、页面） |
| 弱引用 | 下一次 GC | GC 后返回 null | WeakHashMap、ThreadLocal |
| 虚引用 | 对象回收时入 ReferenceQueue | 永远返回 null | 堆外内存（DirectByteBuffer） |

## 6. G1 原理

```mermaid
graph TB
    subgraph "G1 堆 = 多个 Region"
        R1["Eden<br/>Region"]
        R2["Eden<br/>Region"]
        R3["Survivor<br/>Region"]
        R4["Old<br/>Region"]
        R5["Old<br/>Region"]
        R6["Humongous<br/>Region"]
        R7["Free<br/>Region"]
    end

    subgraph "Remembered Set"
        RS["每个 Region 有 RSet<br/>记录跨 Region 引用<br/>避免全堆扫描"]
    end

    R1 -->|"Minor GC"| R3
    R3 -->|"年龄超阈值"| R4
    R4 -->|"Mixed GC<br/>回收价值高的 Old Region"| R7
```

G1 GC 周期：
1. **Young GC** — 回收所有 Eden Region
2. **并发标记** — 与用户线程并发确定存活对象
3. **Mixed GC** — 回收部分 Old Region（按回收价值排序）
4. **Full GC** — Mixed GC 跟不上时触发（最差情况，单线程）

## 8. JVM 调优参数速查

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `-Xms` / `-Xmx` | 堆初始/最大大小 | `-Xms4g -Xmx4g` |
| `-Xss` | 线程栈大小 | `-Xss256k` |
| `-XX:MetaspaceSize` | 元空间初始大小 | `256m` |
| `-XX:MaxMetaspaceSize` | 元空间最大大小 | `512m` |
| `-XX:+UseG1GC` | 使用 G1 收集器 | - |
| `-XX:MaxGCPauseMillis` | 期望最大停顿 | `200` |
| `-XX:+UseZGC` | 使用 ZGC | - |
| `-XX:+HeapDumpOnOutOfMemoryError` | OOM 自动 dump | - |
| `-XX:HeapDumpPath` | dump 文件路径 | `/data/logs/` |
| `-XX:+DisableExplicitGC` | 禁用显式 GC | - |
