## FST(Finite State Transducer) vs Trie

```mermaid
graph TD
    subgraph "Trie 树 (不压缩)"
        T_ROOT["root"] --> T_M["m"]
        T_M --> T_O1["o"]
        T_O1 --> T_N1["n(1)"]
        T_N1 --> T_D1["d"]
        T_D1 --> T_A1["a"]
        T_A1 --> T_Y1["y(2)"]
        T_N1 --> T_T1["t"]
        T_T1 --> T_H1["h(3)"]
        T_O1 --> T_O2["o"]
        T_O2 --> T_N2["n(4)"]
    end

    subgraph "FST (前缀压缩)"
        F_ROOT["root"] --> F_M["m"]
        F_M --> F_O["o"]
        F_O --> F_N["n/1"]
        F_N --> F_BRANCH["分支"]
        F_BRANCH --> F_DAY["day/2"]
        F_BRANCH --> F_TH["th/3"]
        F_O --> F_ON["on/4"]
    end

    style T_ROOT fill:#FFE4B5
    style F_ROOT fill:#90EE90
```

| 对比 | Trie | FST |
|------|------|-----|
| 存储结构 | 树状,每节点存字符 | 图状,共享前缀+后缀 |
| 内存占用 | 较大 | 极小(可全加载到内存) |
| ES 用途 | Term Index | Term Dictionary |

## 搜索流程

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant BF as BloomFilter
    participant Dict as Term Dictionary(FST)
    participant PL as Posting List

    Client->>BF: 1.查询 Term="elasticsearch"
    alt BloomFilter 返回 false
        BF-->>Client: 不存在,直接返回空
    else BloomFilter 返回 true
        BF-->>Client: 可能存在,继续查询
        Client->>Dict: 2.词典查找 TermId
        Dict-->>Client: TermId=42
        Client->>PL: 3.获取Posting List
        PL-->>Client: DocId=[1,3,5,7,9]
    end
```

## 跳表加速 Posting List 求交

```mermaid
graph TD
    subgraph "Posting List A (带跳表)"
        A1["1"] --> A3["3"] --> A5["5"] --> A7["7"] --> A9["9"] --> A11["11"] --> A13["13"] --> A15["15"]
        A1 -.->|"skip"| A5
        A5 -.->|"skip"| A9
        A9 -.->|"skip"| A13
    end

    subgraph "Posting List B"
        B1["3"] --> B2["6"] --> B3["7"] --> B4["9"] --> B5["12"] --> B6["15"]
    end

    INTER["求交结果: [3,7,9,15]"]

    A1 -.-> INTER
    B1 -.-> INTER

    style INTER fill:#90EE90
```

**关键**: 跳表每 3 个元素建一个跳跃指针,当 A 的当前值 < B 的当前值时,利用跳跃指针快速跳过不可能重合的区间,避免逐元素比较。
