## 3. Trigger 触发流程

```mermaid
flowchart TD
    TRIGGER["Trigger 组合条件"]

    TRIGGER --> CT["CountTrigger<br/>count >= N?"]
    TRIGGER --> ET["EventTimeTrigger<br/>watermark >= window.end?"]

    CT --> |"YES → FIRE"| CALC["执行窗口计算<br/>AggregateFunction"]
    ET --> |"YES → FIRE"| CALC

    CT --> |"NO → CONTINUE"| WAIT["继续收集数据"]
    ET --> |"NO → CONTINUE"| WAIT

    CALC --> PURGE["Purge 窗口状态<br/>FIRE_AND_PURGE"]
    CALC --> OUTPUT["输出: WindowResult<br/>window+key+sum+count"]
```

## 4. SideOutput 延迟数据处理

```mermaid
flowchart TD
    DATA["数据到达"] --> CHECK{"watermark ><br/>window.end +<br/>allowedLateness?"}

    CHECK --> |"否 → 正常"| WINDOW["进入窗口计算"]
    CHECK --> |"是 → 延迟"| SIDE["SideOutput<br/>侧输出流"]

    WINDOW --> MAIN["主流输出<br/>核心业务结果"]
    SIDE --> LATE["延迟流输出<br/>补偿/告警/日志"]

    MAIN --> RESULT["结果确定性"]
    LATE --> RESULT2["延迟数据不丢失"]
```

## 5. Tumbling vs Sliding 数据分配对比

```
Tumbling Window (size=5s):
─────────────────────────────────────────────
[0──────5)     [5──────10)    [10─────15)
   ↑               ↑              ↑
 t=1,2,3        t=6,7         t=12

每条数据只属于 1 个窗口


Sliding Window (size=10s, slide=5s):
─────────────────────────────────────────────
[0─────────10)
     [5──────────15)
          [10─────────20)
               [15──────────25)
   ↑     ↑        ↑
 t=6 属于 2 个窗口: [0,10) 和 [5,15)

数据在多个窗口中重复计算！

窗口数 = floor((timestamp + slide) / slide)
```

## 6. 核心类关系

| 类 | 职责 | 触发条件 |
|----|------|----------|
| `WindowAssigner` | 决定数据分配到哪些窗口 | 每条数据到达时 |
| `Trigger` | 决定窗口何时触发计算 | Watermark + Count + 定时器 |
| `WindowOperator` | 组合 Assigner + Trigger 处理数据 | 数据流持续 |
| `SideOutputCollector` | 收集延迟数据 | 超过 allowedLateness |
| `WindowResult` | 窗口聚合结果 | Trigger 触发后 |
